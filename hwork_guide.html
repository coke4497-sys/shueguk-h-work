스프레드시트 안의 시트 두 개: HWORK목록(출제한 숙제 정답 보관), 제출기록(학생 제출 자동 저장). 둘 다 코드가 자동으로 관리하니 직접 손댈 필요는 없습니다.

새 H WORK 출제하기 (매주 하는 일)

  1. 슈국 티쳐스 → H WORK → 출제 를 엽니다.
  2. 담당 강사 · 제목 · 문항 수(20~70)를 입력하고, 학교 목록에 해당 학교들을 추가합니다.
  3. 문항마다 유형(5지선다 · OX · 서술형)을 고르고 정답을 입력합니다. 위쪽 「모든 문항 유형 일괄」 버튼으로 빠르게 세팅한 뒤 일부만 바꿔도 됩니다.
  4. 아래 「시트에 저장」을 누릅니다. 초록색 "저장 완료 ✓"가 뜨면 등록 완료입니다.
이게 전부입니다. 학생 화면을 새로고침하면, 학생이 담당 강사 → 제목을 고를 때 방금 만든 숙제가 자동으로 나타납니다. 같은 강사·제목으로 다시 저장하면 덮어쓰기 됩니다(수정 가능).

선택한 학생에게 과제 배정하기 (배정·현황)

특정 학생들에게 과제를 배정하고, 누가 제출/미제출했는지 한눈에 봅니다. (선택 기능 — 안 써도 출제·제출은 그대로 동작합니다.)
  1. 슈국 티쳐스 → H WORK → 배정·현황 을 엽니다. 비밀번호 101011.
  2. 위에서 담당 강사 → 제목으로 배정·확인할 과제를 고릅니다.
  3. 학생 배정에서 대상을 고릅니다 — 전 학년 / 학년 / 개인 / 일부.
    ('일부'에서는 학년·학교로 거른 뒤 「보이는 학생 모두 선택」을 누르면 특정 학년·학교 학생만 한 번에 담을 수 있습니다.)
  4. 「이 학생들에게 배정」을 누르면 아래 배정 내역에 추가됩니다. 여러 번 배정하면 합쳐지고, 각 배정은 삭제할 수 있습니다.
  5. 제출 현황 → 「현황 불러오기」 → 배정 · 제출 · 미제출 인원과 명단을 확인합니다.
배정하면 학생 개인 페이지의 'H-work' 칸에 그 학생에게 배정된 과제만 표시됩니다.
제출 대조는 이름+학교 기준입니다. 같은 학교에 동명이인이 2명 이상이면 자동 구분이 어려워 ⚠️ 표시가 뜨며, 그땐 학생ID로 직접 확인하세요. ·· 배정 데이터는 H WORK 시트가 아니라 지필고사 리포트 스프레드시트의 '배정' 탭(학생 명단·공지와 같은 곳)에 저장됩니다.

서술형(키워드) 채점 규칙 — 중요

서술형 정답은 다음 규칙으로 자동 채점됩니다(공백·영문 대소문자는 무시).
  • 쉼표(,) = 나눈 키워드를 모두 포함해야 정답 (순서는 상관없음).
    예: 전차, 개찰구 → 두 단어가 모두 있어야 정답.
  • 빗금(/) = 같은 자리의 대체 표현(하나만 맞아도 정답).
    예: 은유/메타포 → 둘 중 하나면 정답.
  • 조합도 가능: 전차/전차들, 개찰구 → (전차 또는 전차들) 그리고 개찰구.
점수는 항상 맞은 문항 / 총 문항으로 계산됩니다(배점 없음). 학생에게는 "정답 키워드를 쉼표로 구분해 적으라"고 안내하면 채점이 더 정확합니다.

학생 / 강사 사용법

학생 — 학생 제출용 주소 접속 → 담당 강사 → 제목 선택 → 학교(목록에서 고르거나 「명단에 없음」 후 직접 입력) → 학년(중1~고3) → 이름 입력 → 답안 체크 → 제출 → 결과(맞은 개수·문항별 정오) 즉시 확인.

강사 — 슈국 티쳐스 → H WORK → 확인 → 담당 강사 선택(선택해야 목록이 보입니다) → 제목·학교·학년·이름으로 더 좁히기 → 학생을 누르면 채점 결과 조회. 표 위에 제출 건수·평균 정답률도 표시됩니다.

화면이나 코드를 고쳤을 때

화면(HTML)을 고쳤을 때 — 고친 파일을 GitHub 저장소에 같은 이름으로 다시 업로드(Add file → Upload files → Commit) 후 1~2분 기다렸다 새로고침합니다. 옛 화면이 남아 있으면 강력 새로고침(Ctrl+Shift+R)이 도움이 됩니다.

채점 코드(Code.gs)를 고쳤을 때 — 코드 수정은 바로 반영되지 않습니다. Apps Script에서 배포 → 배포 관리 → (연필) 편집 → 버전: 「새 버전」 → 배포로 다시 배포하세요. 이렇게 하면 주소(API)가 그대로라 화면을 안 고쳐도 됩니다.

만약 「새 배포」로 만들어 API 주소가 바뀌면, 세 HTML 파일 맨 위 const API_URL = "..." 의 주소를 새 주소로 바꿔 저장한 뒤 GitHub에 다시 올려야 합니다. 그리고 이 문서의 「채점 서버」 칸도 갱신하세요.

문제가 생기면

  • 출제한 숙제가 학생 화면에 안 보일 때 — 출제 앱에서 「시트에 저장」 후 초록색 완료 메시지가 떴는지 확인하고, 학생 화면을 새로고침합니다.
  • 목록이 아예 안 뜨고 "서버 연결 실패"가 뜰 때 — 인터넷 연결을 확인하고, 그래도 안 되면 위 「코드를 고쳤을 때」를 참고해 API 주소가 맞는지 확인합니다.
  • 점수가 이상할 때 — 출제 앱에서 그 숙제의 정답·유형을 확인하고, 서술형이면 위 채점 규칙(쉼표=모두, 빗금=택일)에 맞게 적혔는지 확인한 뒤 다시 저장합니다. (이미 제출된 기록은 소급되지 않으니, 확인은 다시 제출해 비교하세요.)
  • "확인되지 않은 앱" 경고(Apps Script 배포 시) — 본인이 만든 앱이라 나오는 정상 경고입니다. 고급 → 계속으로 진행합니다.
